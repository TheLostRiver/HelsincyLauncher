## AT-053 — facade.start_download 真实入队

**范围**: 实现 `DownloadFacade::start_download()` 真正入队任务到 JobRuntime

**修改文件**:
- `crates/module-downloads/src/facade/mod.rs`
  - 从 request 提取信息
  - 构建 `JobSnapshot<()>` (注意 extension 为 `None` 因为 kernel-jobs 用的是 `JobSnapshot<()>`)
  - 调用 `job_runtime.create(&snapshot)?` 或类似方法入队
  - 返回 `AcceptedJob { job_id, module, kind, queued_at }`

**验证**:
- 单元测试：call start_download + verify job appears in snapshot store
- smoke gates 继续通过

**实现代码模式**:
```rust
pub fn start_download(&self, request: StartDownloadRequestDto) -> AppResult<AcceptedJob> {
    let job_id = JobId::generate();
    let snapshot = JobSnapshot {
        job_id: job_id.clone(),
        module: "downloads".to_string(),
        kind: "download".to_string(),
        state: JobState::Queued,
        ui_state: JobUiState::Queued,
        progress: JobProgress::default(),
        recoverable: true,
        updated_at: IsoDateTime::now(),
        extension: None,
    };
    self.deps.job_runtime.create(&snapshot)?;
    Ok(AcceptedJob { 
        accepted: true,
        job_id,
        module: "downloads".to_string(),
        kind: "download".to_string(),
        queued_at: IsoDateTime::now(),
    })
}
```

## 关键约束

- **kernel-jobs** 中 `JobSnapshot<E>` 目前只用 `JobSnapshot<()>` (E = 单元类型)
- 模块级别的 checkpoint 不属于 kernel-jobs，应留在 module facade 内
- DownloadJobDriver restore 需要能访问 checkpoint store（需要从 facade 拿）
- Module facade 必须通过 composition-root 依赖注入 job_runtime
